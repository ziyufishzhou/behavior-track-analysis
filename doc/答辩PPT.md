# 研究背景与问题

- 啮齿动物行为实验常用于神经科学和药理学研究
- DeepLabCut 可以输出动物关键点坐标，但后续分析仍依赖大量人工处理
- 实验流程中存在文件多、标签混乱、CSV 格式不统一、ROI 标注分散等问题
- 本项目希望把“姿态估计结果”进一步转化为可复用的行为指标和统计图表

---

# 系统建设目标

- 支持 OF、EPM、TCT 三类常见行为实验
- 实现视频导入、标签管理、DLC CSV 预处理、ROI 标注、指标计算和统计绘图
- 提供 Web 工作台，降低命令行脚本使用门槛
- 提高实验数据处理的自动化、规范性和可重复性

# DLC 模型训练与评估

- 模型骨干网络：ResNet-50
- 训练集比例：95%，Shuffle=1，训练轮数：830 epochs
- 测试集 RMSE：3.27 px
- pcutoff=0.6 后测试集 RMSE：2.87 px
- 测试集 mAP：59.12，mAR：70.10
- 训练集与测试集误差接近，模型泛化表现较稳定

---

# 核心功能实现

- 视频与标签管理：支持可配置视频目录，保存 Experiment、Group、Condition、MouseID、Phase
- CSV 预处理：收集 DLC 输出、修复表头、低置信度过滤、短缺失插值
- ROI 标注：为 OF、EPM、TCT 保存实验区域配置
- 行为分析：计算中心区时间、开放臂时间、社交偏好指数等指标
- 统计绘图：输出 summary 表格、轨迹图、热力图和统计图

---

# Web 工作台

- 项目状态页：集中展示视频、CSV、metadata、ROI 和分析结果状态
- 导入视频页：支持切换视频根目录，避免写死本地路径
- 预处理页：一键运行 CSV 收集、修复、分组和 metadata 生成
- 数据分析页：统一配置 FPS、置信度阈值和分析时长
- 绘图页：支持图表类型、颜色方案和统计检验配置

---

# 工程规范与可维护性

- 目录分层：`web/`、`preprocessing/`、`experiments/`、`plotting/`、`config/`、`doc/`
- 配置分离：路径和环境配置不写死在业务代码中
- 环境复现：提供 `requirements.txt` 和 `environment.yml`
- 启动简化：使用 `python start_web.py` 启动系统
- 健康检查：通过 `scripts/verify_project.py` 验证核心模块和接口
