### 三、项目结构与流程整理

已对项目流程进行梳理，当前系统主要包括：

- 数据层：原始视频、DLC 输出 CSV、ROI 标注文件、实验元数据。
- 预处理层：CSV 收集、CSV 修复、标签分组、元数据匹配。
- 分析层：OF、EPM、TCT 三类实验指标计算。
- 绘图层：统计图、轨迹图、热力图和汇总表导出。
- 交互层：基于 Flask 的 Web 操作界面。

### 四、代码修改摘要

已完成的主要代码修改包括：

- 修正 `experiments.OF`、`experiments.EPM`、`experiments.TCT` 等模块导入路径。
- 新增和完善 Web 端预处理接口，包括 CSV 收集、CSV 修复、按标签分组、元数据处理和一键预处理。
- 重写 CSV 收集逻辑，使系统可以从 `video/` 目录递归收集 DeepLabCut 生成的 CSV 文件到 `data/raw_csv`。
- 改进 CSV 修复逻辑，兼容 DeepLabCut 2.x/3.x 常见表头格式，并支持低置信度点过滤和短缺失插值。
- 新增 `experiments/common.py`，统一 ROI、CSV 和 bodypart 列读取逻辑。
- 改进 OF、EPM、TCT 三类实验分析脚本的指标输出。
- 改进元数据匹配逻辑，使视频名和 `*_result.csv` 能够更稳定对应。
- 改进 TCT 阶段识别逻辑，优先使用元数据，其次根据文件名中的 S/N/H 等标记判断。
- 优化绘图逻辑，使对照组优先显示，并处理样本量过小、零方差、空值等统计边界情况。
