# 工程规范整改清单

本文档按照“本科毕业设计项目的工程规范”逐项记录整改情况。

## 1. 目录结构规范

目标：代码、数据、输出、论文资料分开放置。

当前状态：

- `web/`：Web 工作台。
- `preprocessing/`：CSV 预处理。
- `experiments/`：OF、EPM、TCT 实验分析。
- `plotting/`：统计绘图。
- `config/`：路径、设置和配置示例。
- `scripts/`：项目检查和辅助脚本。
- `doc/`：论文、图表、参考文献和项目记录。
- `data/`、`video/`、`output/`、`models/`：本地数据和生成文件，不建议提交。

已完成：

- 论文相关资料统一放入 `doc/`。
- 新增 `scripts/verify_project.py`。
- 新增 `.gitignore` 忽略数据、输出、模型和缓存文件。

## 2. 路径不要写死

目标：换电脑或换数据目录时，不需要修改源码。

已完成：

- 视频根目录改为可配置，默认使用 `video/`。
- Web 页面可以保存视频根目录。
- DLC Python 路径改为可配置。
- Web 启动器不再在源码中绑定固定 Python 绝对路径。
- 新增 `.env.example` 和 `config/app_settings.example.json`。

## 3. 配置和代码分开

目标：参数和路径集中管理，业务代码只读取配置。

已完成：

- 实验参数集中在 `config.py`。
- 路径和应用设置集中在 `config/paths.py`。
- Web 本地设置保存到 `data/app_settings.json`。
- 配置示例放在 `config/app_settings.example.json`。

## 4. 一个功能一个模块

目标：不同职责放在不同模块中，降低维护难度。

当前模块：

- `preprocessing/collect_csv.py`：收集 DLC CSV。
- `preprocessing/fix_csv.py`：修复 DLC CSV。
- `preprocessing/group_csv.py`：按 metadata 分组。
- `preprocessing/build_metadata.py`：生成或更新 metadata。
- `experiments/*/analyze.py`：实验指标计算。
- `experiments/*/roi_tool.py`：ROI 标注。
- `plotting/`：绘图和统计。
- `web/routes/`：Web API。

## 5. 函数名字要清楚

目标：通过函数名能看懂主要行为。

已完成：

- 使用 `collect_dlc_csv()`、`group_by_metadata()`、`load_roi_regions()`、`get_video_dir()`、`set_video_dir()` 等清晰命名。
- 新增 `scripts/verify_project.py` 中的检查函数，如 `check_imports()`、`check_paths()`、`check_web_endpoints()`。

后续建议：

- 继续清理旧 GUI 代码中较宽泛的函数名。

## 6. 错误提示要清楚

目标：失败时告诉用户“哪里错了、怎么改”。

已完成：

- Web 请求失败时会显示后端返回的具体错误。
- 视频目录不存在时明确提示路径。
- DLC Python 未配置或不存在时明确提示。
- 状态页展示视频、CSV、ROI、metadata 和 summary 是否就绪。

## 7. 依赖要能复现

目标：别人可以根据依赖文件创建运行环境。

已完成：

- `requirements.txt`：pip 依赖列表。
- `environment.yml`：Conda 环境配置。
- README 中加入环境创建命令。

## 8. 启动方式要简单

目标：尽量一条命令或双击启动。

已完成：

- 推荐使用 `python start_web.py`。
- 保留 `start_web_behavioranalyze.bat` 作为 Windows 双击入口。
- 启动器会检查端口、启动服务并打开浏览器。

## 9. 重要流程要能被测试

目标：不是只说“能跑”，而是有检查脚本证明主要入口可用。

已完成：

- 新增 `scripts/verify_project.py`。
- 检查内容包括核心模块导入、路径配置、Web 关键接口、metadata 字段。

运行方式：

```powershell
python scripts/verify_project.py
```

## 10. 文档要告诉别人怎么用

目标：README 和项目文档能指导安装、启动、使用和答辩演示。

已完成：

- 重写 `README.md`。
- 新增 `doc/项目演示流程.md`。
- 持续维护 `doc/项目修改记录.md`。
- 新增本文档 `doc/工程规范整改清单.md`。

## 当前结论

项目已经从“自己电脑上能跑的脚本集合”逐步整理为“有入口、有配置、有状态、有检查、有文档”的毕业设计工程项目。
