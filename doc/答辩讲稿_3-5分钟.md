# 毕业设计答辩讲稿（3-5分钟）

题目：基于 DeepLabCut 的啮齿动物行为分析系统设计与实现

## 第1页 封面

各位老师好，我的毕业设计题目是《基于 DeepLabCut 的啮齿动物行为分析系统设计与实现》。本项目主要面向啮齿动物行为实验的数据处理流程，在 DeepLabCut 姿态估计结果的基础上，实现视频管理、数据预处理、ROI 标注、行为指标计算和统计绘图等功能。

## 第2页 研究背景与问题

啮齿动物行为实验常用于神经科学、药理学等研究场景。DeepLabCut 可以自动识别动物关键点，并输出坐标数据，但它主要解决的是姿态估计问题。实际实验中，研究人员还需要继续处理大量 CSV 文件，维护实验标签，进行 ROI 标注，再计算 OF、EPM、TCT 等实验指标。这个过程如果完全依赖人工和零散脚本，容易出现路径混乱、文件对应错误、格式不一致和结果难以复现等问题。因此，我希望设计一个完整的行为分析系统，把姿态估计结果进一步转化成可用的行为学结果。

## 第3页 系统建设目标

本系统的目标主要有四点。第一，支持 OF、EPM、TCT 三类常见行为实验。第二，实现从视频导入、标签设置、DLC CSV 预处理、ROI 标注、指标计算到统计绘图的一体化流程。第三，提供 Web 工作台，让用户不需要频繁操作命令行脚本。第四，提高数据处理的自动化程度、规范性和可重复性，使项目不仅能在我自己的电脑上运行，也能更容易迁移和维护。

## 第4页 系统总体架构

这一页是系统总体架构。系统采用分层模块化设计，主要分为数据层、预处理层、分析层、绘图层和交互层。数据层包括原始视频、DLC 输出 CSV、ROI 配置和 metadata。预处理层负责 CSV 收集、修复和分组。分析层分别实现 OF、EPM、TCT 三类实验的指标计算。绘图层负责生成 summary 表格、轨迹图、热力图和统计图。最上层是 Flask Web 工作台，用于统一操作这些功能。

## 第5页 数据处理流程

这一页展示的是系统数据流程。用户首先导入视频并设置实验标签，然后运行 DeepLabCut 得到 CSV 文件。之后系统会收集 CSV，修复 DLC 表头和低置信度点，再根据 metadata 按实验类型和标签分组。完成 ROI 标注后，系统根据不同实验规则计算行为指标，最后输出统计图和结果文件。这个流程对应真实实验中的数据处理链路，也方便在答辩时逐步演示。

## 第6页 DLC 模型训练与评估

本项目使用 ResNet-50 作为 DeepLabCut 模型的骨干网络进行训练。模型训练集比例为 95%，shuffle 为 1，训练轮数为 830 epochs。从评估结果看，测试集 RMSE 为 3.27 像素，在 pcutoff 等于 0.6 的置信度筛选后，测试集 RMSE 降低到 2.87 像素。训练集和测试集误差非常接近，说明模型没有明显过拟合，具有比较稳定的泛化能力。测试集 mAP 为 59.12，mAR 为 70.10，能够为后续行为指标计算提供可靠的关键点坐标输入。

## 第7页 核心功能实现

系统核心功能包括五部分。第一是视频和标签管理，支持配置视频根目录，并保存 Experiment、Group、Condition、MouseID 和 Phase 等信息。第二是 CSV 预处理，能够兼容不同 DLC 输出格式，并进行低置信度过滤和短缺失插值。第三是 ROI 标注，针对 OF、EPM、TCT 保存不同区域配置。第四是行为分析，例如 OF 的中心区时间、EPM 的开放臂时间、TCT 的社交偏好指数等。第五是统计绘图，用于生成 summary 表格和可视化结果。

## 第8页 Web 工作台

为了降低使用门槛，我实现了一个 Web 工作台。现在系统包含项目状态、导入视频、DLC 分析、预处理、ROI 标注、数据分析和绘图设置等页面。其中项目状态页可以直接显示视频目录、CSV 数量、metadata、ROI 和 summary 是否就绪。导入视频页支持切换视频根目录，避免路径写死。预处理页可以一键运行 CSV 收集、修复、分组和 metadata 生成。这样用户可以按实验流程逐步操作，也更适合现场演示。

## 第9页 工程规范与可维护性

除了功能实现，我也对项目做了工程化整理。目录上分为 web、preprocessing、experiments、plotting、config 和 doc 等模块。路径和环境配置不再写死在业务代码中，而是通过配置文件和 Web 页面设置。依赖方面提供了 requirements.txt 和 environment.yml。启动方面使用 python start_web.py。项目还提供 scripts/verify_project.py 健康检查脚本，用于检查核心模块、路径配置、Web 接口和 metadata 字段。这些工作主要是为了让项目从“自己能跑”变成“别人也能比较顺利地运行和理解”。

## 第10页 测试与运行结果

目前项目已经完成核心验证。Python 模块编译和导入检查通过，Web 首页和状态接口都能正常返回 HTTP 200，metadata 字段结构检查通过。系统可以识别当前视频目录、CSV 数量、ROI 状态和 summary 输出。答辩演示时，可以按照“导入视频、预处理、ROI、数据分析、统计绘图”的流程展示。如果 DLC 分析耗时较长，也可以提前准备好 DLC 输出 CSV，现场重点展示系统流程和结果生成。

## 第11页 总结与展望

总结来说，本项目完成了一个面向啮齿动物行为实验的 Web 化数据分析工作台，把 DeepLabCut 的姿态估计结果进一步转化为行为学指标和统计结果。系统支持多实验类型、可配置路径、状态检查和标准化输出。后续还可以继续扩展更多行为实验范式，加入批量数据质控，并进一步完善多用户管理和更丰富的统计分析功能。我的汇报结束，请各位老师批评指正。
