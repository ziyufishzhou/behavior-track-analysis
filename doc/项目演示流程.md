# 项目演示流程

本文档用于毕业设计答辩或项目检查时快速演示系统功能。

## 一、启动系统

在项目根目录运行：

```powershell
python start_web.py
```

浏览器打开：

```text
http://127.0.0.1:5000
```

## 二、检查项目状态

进入“项目状态”页面，确认：

- 视频目录存在。
- 视频数量大于 0。
- metadata 文件存在或准备生成。
- ROI 文件状态明确。
- CSV 和 summary 输出状态明确。

## 三、导入视频并设置标签

进入“导入视频”页面：

1. 设置视频根目录。
2. 点击“刷新视频列表”。
3. 选择一个或多个视频。
4. 填写 Experiment、Group、Condition、MouseID、Phase。
5. 点击“应用标签到选中”。
6. 点击“保存并分组”或保存 metadata。

## 四、DLC 分析

进入“DLC 分析”页面：

1. 设置 DLC 模型路径。
2. 设置 DLC Python 路径。
3. 刷新待分析视频列表。
4. 选择少量演示视频，或不选中表示分析全部。
5. 点击“开始 DLC 分析”。

答辩演示时，如果 DLC 分析耗时较长，可以提前准备好 DLC 输出 CSV，现场只展示配置和流程。

## 五、CSV 预处理

进入“预处理”页面：

1. 运行“收集 CSV”。
2. 运行“修复 CSV”。
3. 运行“按标签分组”。
4. 运行“生成 metadata”。

也可以点击“运行完整预处理”。

## 六、ROI 标注

进入“ROI 标注”页面：

- 启动 OF ROI 标注工具。
- 启动 EPM ROI 标注工具。
- 启动 TCT ROI 标注工具。

答辩时建议提前完成 ROI 标注，现场展示 ROI 文件已就绪即可。

## 七、数据分析

进入“数据分析”页面：

1. 选择需要分析的实验类型。
2. 检查 FPS、置信度阈值、分析时长。
3. 点击“开始分析”。
4. 观察日志输出。

## 八、统计绘图

进入“绘图设置”页面：

1. 选择实验类型。
2. 点击“自动查找”选择 summary 文件。
3. 选择图表类型、颜色和统计检验方式。
4. 点击“生成图表”。
5. 查看输出图和统计结果。

## 九、建议演示数据

建议提前准备：

- 每类实验 1 到 2 个短视频，或对应的 DLC CSV。
- 已保存的 `data/metadata.xlsx`。
- 已完成的 ROI JSON 文件。
- 至少一个可展示的 summary 结果和统计图。

这样可以保证答辩演示稳定、快速、可控。
